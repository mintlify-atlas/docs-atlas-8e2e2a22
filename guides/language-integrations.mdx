## Overview

Silero VAD's ONNX model can be loaded by any runtime that supports [ONNX Runtime](https://onnxruntime.ai/getting-started). The repository ships community-contributed examples for several languages, all located in the `examples/` directory.

<Info>
These examples are **community-maintained**. They use the ONNX model directly via the ONNX Runtime SDK, and each one implements its own audio I/O and post-processing. APIs and dependencies may differ from the Python package.
</Info>

## Implementing Your Own Integration

If you are integrating with a runtime not listed above, you need to handle three things:

1. **Audio I/O** — load and resample audio to 8000 Hz or 16000 Hz as raw float32 PCM.
2. **State management** — the ONNX model takes a GRU state tensor as input and returns an updated state; you must pass the state from one call to the next.
3. **Post-processing** — threshold comparison, segment boundary detection, and padding logic must be reimplemented in your language.

The ONNX model input/output schema:

| Name | Shape | Dtype | Description |
|------|-------|-------|-------------|
| `input` | `(batch, samples)` | `float32` | Audio chunk (512 samples @ 16 kHz, 256 @ 8 kHz) |
| `state` | `(2, batch, 128)` | `float32` | LSTM hidden state `(h, c)` stacked — zero-initialize on first call |
| `sr` | scalar | `int64` | Sampling rate (`8000` or `16000`) |
| **output** | `(batch, 1)` | `float32` | Speech probability for this chunk |
| **stateN** | `(2, batch, 128)` | `float32` | Updated LSTM state (feed back into next call) |

Refer to the existing community examples for concrete patterns — the C++ and Rust examples are the most complete references for state management.
